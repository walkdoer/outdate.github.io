使用Node-wekit开发桌面程序总结

发表于 2015-01-26

因为最近在个人项目life-time-tracker中用node-webkit + react来开发桌面程序，中间遇到了一些问题，这里记下来，没什么技术含量；有缘人如果看到的话，这里不适合入门，只言片语的描述一些我遇到的问题而已，文档请移步查看wiki

为什么要用node-webkit来做开发

简单就是：用我熟悉和喜爱的技术来快速迭代开发

  1. 用html+js+css来开发UI界面非常快速，可以利用已有的前端工具或框架，如jquery,react,angular,bootstap等来快速开发用户界面。
    oxs下的32/64系统，可能不同系统有些功能会有区别
  2. 可以直接调用Node模块，这个是我最喜欢的一个特性,可以尽情发挥想象力，不再受限于浏览器。node-webkit = node + webkit
  3. 跨平台，一份代码，进过打包之后可以运行于window,linux,mac

node-webkit 和 React

选择React是因为个人喜欢React背后的一些理念（组件化,Data-flow），借机验证一下React开发较为复杂的界面上，会不会遇到一些问题。

不过使用React的话在调试上面会遇到一些问题，主要是新版node-webkit的问题，后面

构建

使用gulp来进行构建，其中可以使用node-webkit-builder来进行node-webkit打包部署，非常方便。

遇到的几个问题

窗口关闭与退出

点击关闭时，只是关闭窗口，后台继续运行

点击dock(Mac上才有)之后打开程序

JS运行环境的问题

在node_modules中要引用nw.gui模块，不能使用require，会提示找不到模块，正确的用法是使用window.require

问题资料：

  • 可以参考的issue
  • Differences-of-JavaScript-contexts

快捷键（站内与全局快捷键）

全局快捷键，指的是离开程序界面，在操作系统上注册的快捷键。这个可以使用node-webkit提供的API

站内快捷键,这个可以使用DOM监听键盘输入，我在项目中采用的是 mouseTrap,非常简单，具体可以参考mouseTrap的文档。

复制黏贴

要开启复制黏贴的快捷键功能，需要使用node-webkit提供的接口createMacBuiltin, 如果采用自定义的菜单，是无法启用复制黏贴的功能的;

打开外部程序或链接

在node-webkit中，不能直接使用window.open()来打开外部链接，因为会直接使用node-webkit来打开这个链接，正确的用法是调用gui.Shell, 更多参考gui.Shell

Node-webkit的阻塞问题

在开发的过程中遇到了一个问题，每当我进行数据库查询的时候，UI的渲染会被阻塞，造成页面的卡顿现象，很自然的想法就是通过child_process来进行优化, 但是由于node-webkit在child_process上有一些问题，使用起来遇到许多奇奇怪怪的问题，至今我也没有定位到原因，所以就放弃了child_process的解决方案，寻找其他方法。

接着想到通过开启一个新的窗口来运行，避免和原来的主窗口处于同一个render process, 但是事实证明我太天真了，结果还是一样的，因为通过gui.Window.open()打开的新窗口，其实与父窗口是处于同一个render process，UI阻塞的问题依旧存在。

后面通过查找wiki，找到了 new-instance 配置项，最后解决了问题，通过打开一个 new-instance的隐藏window来运行服务端，避免与主窗口处于同一个render process

官方wiki的介绍
